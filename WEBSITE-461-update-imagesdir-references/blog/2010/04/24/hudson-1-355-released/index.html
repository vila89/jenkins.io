Hudson 1.355 Released

Tweet
Published on 2010-04-24 by R. Tyler Croy
  • development
  • feedback
  • just for fun
  • links
  • releases

The release of 1.355 came out earlier this week but I hadn't had the chance to write anything up about it. Of course, the work never stops on Hudson so we almost have 1.356 ready to roll out the door, but then Kohsuke tweeted this:

Because of the data center migration going on, I won't be able to release #hudsonci today.

I won't go into details on some of the infrastructure changes we have lined up just yet, so here's the breakdown of 1.355 #### Bugs fixed <li class=bug> Colored ball image at top of build pages was broken for Hudson in some web containers (fixed by removing workaround for a Firefox bug fixed since 3.0.5/Dec2008). (issue 2341) <li class=bug> Console page while build is running did not wrap lines when viewed in IE. (issue 5869) <li class=bug> Fixed build history to indicate test failure for MavenBuild and MavenModuleSetBuild. <li class=bug> Make dropdownList work in repeatable content, such as a build step. <li class=bug> Fixed a bug where a job created via XML didn't properly receive upstream/downstream computation. (report) <li class=bug> Argument masking wasn't working correctly for commands run on slaves (report) </ul>

Enhancements

    <li class=rfe> Added the slave retention strategy based on a schedule. <li class=rfe> Added to configure charset option of Mailer.

You can go grab the latest .war file straight from hudson-ci.org or if you're using a native package, use your package manager to upgrade.
